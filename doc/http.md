### 发送一次http请求
    1：DNS域名解析(①浏览器缓存②操作系统缓存③hosts文件④NDS服务器) 获取去ip地址
    2：发起TCP三次握手
        ①：SYN=1,ACK=0,seq=x(seq = 0 就代表这是第0号包）,client进入syn_sent状态，客户端等待服务器回复
        ②：SYN=1,ACK=1,seq=y(表示Server 自己的初始序号,如果y=0,则是服务端的发送的第一个包),ack=x+1(期望收到客户端下一个包的序号)
        ③：ACK=1,seq=x+1(客户端自己的第x+1个包),ack=y+1(期望服务端的地几个包)
       可靠性原因：（1，2）让客户端知道与服务的是可连通的，（2，3）让服务端知道与客户端可连通的。
    3：发送http请求
    4：服务端响应http请求
    5：TCP四次挥手
        ①：客户端向服务端发送FIN请求
        ②：服务端恢复了ACK，作为对FIN的应答（表示收到关闭连接的请求，应答之后对端处于终止等待状态，可以接受数据不能发送数据）
        ③：服务端发送FIN请求，表示服务端也关闭连接（服务端收到对端关闭请求之后可能上层需要做一些处理，在本端也关闭之前，仍然可以向客户端发送数据）
        ④：客户端收到服务端关闭连接请求FIN， 发送应答给服务端，四次挥手结束。（实际上这时还有个TIME_WAIT等待状态，linux是等待一分钟，可配置，这时的端口是可复用的）
